Exercice DockerFile

* Création d'un conteneur `riick`: ```shell docker run -d -p 8080:8080 --name riick rick-morty/api ``` * On peut accéder à l'environnant de développement GraphQL a l'adresse [localhost:8080/ide](http://localhost:8080/ide)
* Si un changement est effectué dans le dockerfile, l'image doit être reconstruit pour que le prochain conteneur construit avec cette image ait les bonnes configurations. * Changeons la configuration du port réseaux: 

							...other command

							EXPOSE 9000

							ENV port=9000

							...other command
* Reconstruisons l'image `rick-morty/api`: ```shell docker build -t rick-morty/api . ```
* Nous allons détruire le conteneur riick et le reconstruire avec les nouvelles configurations. ```shell docker rm riick docker run -d -p 9000:9000 --name riick rick-morty/api ``` * Nous pouvons vérifier le fonctionnement en accédant à l'ide GraphQL à l'adresse [localhost:9000/ide](http://localhost:9000/ide)

Docker image avec un volume


							FROM node:12.18.1-alpine

							LABEL Author="Antoine AMARA"

							# Global install yarn package manager and Git (git can be usefull for jest tests framework)
							RUN apk update && apk add yarn git sqlite make

							# Just a little package to easily add a free licence to a project
							RUN npm install --global generate generate-license svgo

							WORKDIR /workspace
							VOLUME /workspace/

							EXPOSE 8080

							ENV port=8080

							RUN yarn install

							CMD [ "yarn", "serve:dev" ]
* Construction de l'image `rick-morty/api`: ```shell docker build -t rick-morty/api . ```
* Maintenant nous passons le code sous la forme d'un volume partager lors du démarrage du conteneur (et donc plus besoin de reconstruire l'image si on change de dossier): ```shell docker run -d -p 9000:9000 -v ./ --name riick rick-morty/api ```
